How to use the Factory Passport page

Factory 4flo is a single-file, local-first project assistant. It turns rough ideas, feature requests, or GitHub issue numbers into a structured Factory Passport run with snapshots, timeline history, interop payloads, credentials, plugins, and optional automations. In plain words: this page helps you take a messy idea and turn it into an organized project plan. It saves the work in your own browser, shows each step, keeps a history, and can connect to tools like GitHub or deployment services when you choose to set them up.

What this page does

Factory 4flo helps you move from “I have an idea” to “I have a structured project record.” Instead of keeping notes, issue links, decisions, and next steps scattered everywhere, the page stores them inside a Factory Passport run.

Start with messy input

Paste an idea, feature request, or GitHub issue numbers.

Break it into steps

The app creates snapshots like context, specification, implementation, and proof.

Keep a record

Timeline history, exports, and share links help you explain the work later.

Quick start

1

Open Factory Passport

Use the left sidebar and choose Factory Passport.

2

Paste your idea

Add a rough idea, task description, or GitHub issue numbers like #5368 #5366.

3

Pick a template

Use 4-Step Factory for normal product work, Security / Upgrade for risk-focused work, or Minimal Custom for a small run.

4

Start the run

Click Start Factory Run. The page creates the first set of snapshots and timeline events.

5

Review, edit, approve, export

Open each snapshot, improve the text, approve what looks right, then export or share the run.

Automations

Optional triggers and actions, such as “when this happens, do that.”

Plugins

Connect model, git, deployment, execution, storage, proxy, or webhook tools.

Factory runs

A Factory run is the full project record. Think of it like a folder for one idea. It contains the starting prompt, every snapshot, decisions, timeline events, links, plugin choices, and export data.

Example input:
“Improve auth security and create a safer login flow. Include GitHub issues #5368 and #5366.”

Snapshots

A snapshot is one checkpoint in the work. It lets you see what the app understood, what it generated, and whether that step is ready to move forward.

Plugins

Plugins are optional connections. The page works locally without them, but plugins let you connect to outside tools when you want more power.

AI Model Plugin

Choose a local or cloud model for assistant work.

Git Provider Plugin

Connect GitHub, GitLab, Bitbucket, or another repository provider.

Deployment Plugin

Connect preview or hosting tools such as Render-style deployment workflows.

Execution / Webhook Plugin

Connect custom REST APIs, webhook actions, proxies, or local command bridges.

Credentials

Credentials are saved in your browser only. Use this area for tokens, API keys, webhook URLs, local bridge URLs, or notes that plugins may need.

Important: Do not put sensitive production secrets into a public copy of this HTML page. For real production use, connect through a backend proxy.

Share and export

Share Run

Creates a link that can carry the run data in the URL hash.

Share Selected

Creates a link focused on the selected snapshot.

Export JSON

Downloads a structured file you can save, send, back up, or import later.

Plain words glossary

Local-first
The page saves your work in your browser first, instead of needing a server before anything works.
Single-file
The whole app can live in one HTML file.
Factory Passport
A structured record of an idea becoming a plan, implementation, and proof.
Snapshot
A checkpoint in the work that can be reviewed, edited, approved, rejected, shared, or exported.
Interop payload
A clean package of data another tool can understand.
Plugin
A connection to another tool, such as an AI model, GitHub, a deployment service, or a webhook.
Automation
A saved rule that describes what should happen when something else happens.

Tips

  • Start with messy notes. The point is to organize them.
  • Use Export JSON before resetting a run.
  • Use plugins only when you need outside tools.
  • Keep sensitive tokens out of public demos.
  • Use the tooltip beside the definition when explaining the page to non-technical people.
